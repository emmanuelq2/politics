# French Political Pulse — 2027 Presidential Election Poll

A FastAPI web application that polls users on 4 major political fault lines in France, assigns them a political profile, and cross-checks their answers for internal consistency (anti-troll engine).

## 1. Overview

**Goal:** Each visitor votes on 4 defining political issues, receives a political profile label, and gets a consistency score (0–100) that flags internal contradictions in their answers.

**Key design principles:**
- Results hidden until after voting (prevents anchoring)
- One vote per device (cookie-based, 1-year TTL)
- Answers cross-checked against 13 contradiction pairs
- Q10 (political alignment) is a calibration trap: declared party is compared against positions expressed in Q1–Q9
- Submission speed monitored (< 25 seconds for 10 questions = likely bot)

## 2. Architecture

```
POLITICS/
├── main.py              — FastAPI routes (ballot, vote, results, reset, API endpoints)
├── questions.py         — All question configs, political profiles, contradiction pairs, per-candidate calibration ranges
├── scoring.py           — Consistency engine, profile assignment (Euclidean distance), credibility scoring
├── legifrance.py        — Légifrance/PISTE API client (OAuth2, search, 24h cache)
├── build_votes.py       — One-time pipeline: downloads AN bulk data → data/candidate_votes.json
├── requirements.txt     — fastapi, uvicorn, jinja2, python-multipart, httpx, python-dotenv
├── .env.example         — Template for PISTE credentials
├── .gitignore           — Excludes .env, voter_ids.json, legifrance_cache.json
├── templates/
│   ├── ballot.html      — 10-question voting form + live Légifrance refs button per question
│   ├── results.html     — Personal profile + axes + AN deputy vote records + aggregate results
├── static/
│   └── style.css        — Dark theme, responsive layout, vote badge styles
└── data/
    ├── votes.json           — Aggregate vote counts, dimension score arrays, credibility scores
    ├── voter_ids.json       — Set of voter UUIDs (gitignored)
    ├── legifrance_cache.json — Cached Légifrance API responses (gitignored, 24h TTL)
    ├── candidate_votes.json  — Pre-built AN roll-call vote table (generated by build_votes.py)
    └── scrutins_cache/       — Cached AN scrutin ZIP files (9.6 MB leg 16, 20.3 MB leg 17)
```

### Data flow

```
GET /          → ballot.html (check cookie → show ballot or redirect to results)
POST /vote     → validate form → run full_analysis() → persist → set cookie → redirect
GET /results   → decode last_analysis cookie + load candidate_votes.json → show full results page
POST /reset    → clear votes.json and voter_ids.json

GET  /api/references/{question_id}  → asyncio.to_thread(legifrance.fetch_live_references)
POST /api/cache/clear               → asyncio.to_thread(legifrance.clear_cache)
```

### Analysis stored in cookie

After voting, the personal analysis (profile, dimension scores, contradictions, credibility) is base64-encoded JSON stored in a `last_analysis` cookie (1-hour TTL). This avoids re-querying the database on the results page and keeps the architecture stateless per-request.

## 3. Running the App

```bash
pip install -r requirements.txt

# (Optional) Set up Légifrance PISTE credentials
cp .env.example .env
# Edit .env with your PISTE_CLIENT_ID, PISTE_CLIENT_SECRET, PISTE_API_KEY

# (Optional) Pre-build the AN deputy vote table (one-time, ~30s)
python build_votes.py

# Start the server
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Open: **http://localhost:8000**

To reset all votes (also available via UI button):
```bash
curl -X POST http://localhost:8000/reset
```

To clear the Légifrance API cache:
```bash
curl -X POST http://localhost:8000/api/cache/clear
```

## 5. Consistency & Anti-Troll Engine

### Credibility score (0–100)

Starts at 100, reduced by:

| Penalty | Condition |
|---------|-----------|
| −30 | **Major contradiction** (defined pairs, see below) |
| −15 | **Moderate contradiction** |
| −15 per axis | **Party calibration mismatch** (declared candidate ≠ positions on that dimension) |
| −20 | **Submission speed** < 25 seconds for 10 questions |

**Thresholds:**
- ≥ 65 → Consistent profile (green)
- 40–64 → Suspicious / some inconsistencies (orange)
- < 40 → Likely troll or very incoherent (red)

### 13 Contradiction pairs

| Q1 answer | Q2 answer | Severity | Explanation |
|-----------|-----------|----------|-------------|
| `eco_activism=vanguard` | `eco_economy=growth` | major | Eco-hero + unlimited growth |
| `eco_activism=terrorist` | `eco_economy=degrowth` | major | Anti-activist + degrowth |
| `wealth_tax=trickle_down` | `intersectionality=intersectional` | moderate | Trickle-down + class struggle worldview |
| `wealth_tax=zucman_plus` | `intersectionality=traditional` | major | Radical redistribution + traditional values |
| `ukraine_support=troops` | `ukraine_origins=russia_defending` | major | Send troops + Russia is victim |
| `ukraine_support=troops` | `ukraine_origins=nato_proxy` | major | Send troops + NATO proxy war framing |
| `ukraine_support=no_support` | `ukraine_origins=illegal_invasion` | major | Recognise invasion + zero support |
| `systemic_racism=denial` | `lbd_armament=ban_now` | major | Deny police violence + ban LBDs |
| `systemic_racism=systemic` | `lbd_armament=more_force` | major | Systemic racism + more firepower |
| `wealth_tax=trickle_down` | `state_role=planification` | major | Trickle-down + state planning |
| `wealth_tax=zucman_plus` | `state_role=sovereign_state` | moderate | Radical redistribution + RN state exclusion |
| `eco_economy=degrowth` | `eco_activism=terrorist` | moderate | Degrowth + oppose its main advocates |
| `systemic_racism=denial` | `intersectionality=intersectional` | moderate | Deny racism + embrace intersectionality |

### Political profile assignment

9 profiles (+ Abstentionniste) assigned by **Euclidean distance** on 4 dimension axes:

| Profile | eco | econ | ukraine | police |
|---------|-----|------|---------|--------|
| Radical Left | 4.0 | 4.0 | 2.5 | 4.0 |
| LFI / NFP | 3.0 | 4.0 | 2.0 | 4.0 |
| Écologiste | 4.0 | 3.0 | 3.5 | 3.5 |
| Social Democrat | 3.0 | 3.0 | 3.5 | 3.0 |
| Liberal Centrist | 2.0 | 2.0 | 3.5 | 2.0 |
| Conservative Right | 2.0 | 1.5 | 3.0 | 1.5 |
| National Populist (RN) | 1.5 | 2.0 | 1.5 | 1.0 |
| Hard Right | 1.0 | 1.0 | 2.0 | 1.0 |
| Abstentionniste | 2.5 | 2.5 | 2.5 | 2.5 |

**Abstentionniste** is only assigned when the voter explicitly declared Q10=`none` AND no dimension score deviates more than 1.2 from 2.5 (genuinely centrist). A radical who clicks "none" on Q10 still gets their actual profile.

## 9. Légifrance PISTE API Integration

Each question's context block on the ballot has a "⚡ Load live Légifrance references" button that fetches live documents on demand from the official PISTE gateway.

### Setup

1. Register at https://piste.gouv.fr/registration
2. Create an application, subscribe it to the **Légifrance v2.4.2** API
3. Copy `.env.example` to `.env` and fill in credentials:

```env
PISTE_CLIENT_ID=your_client_id
PISTE_CLIENT_SECRET=your_client_secret
PISTE_API_KEY=your_api_key
PISTE_SANDBOX=false
PISTE_CACHE_TTL=86400
```

### How it works

- `legifrance.py` handles OAuth2 Client Credentials flow to `https://oauth.piste.gouv.fr/api/oauth/token`
- Token is reused until 60 seconds before expiry (thread-safe via `threading.Lock`)
- Search endpoint: `POST https://api.piste.gouv.fr/dila/legifrance/lf-engine-app/search`
- Each question maps to a `fond` (JORF or LEGI) + search query in `QUESTION_QUERIES`
- Results cached to `data/legifrance_cache.json` for 24 hours
- Falls back gracefully (button hidden) if PISTE credentials are absent

### Cache management

```bash
# Clear cache via API
curl -X POST http://localhost:8000/api/cache/clear
```

### Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Button doesn't appear | `PISTE_CLIENT_ID` / `PISTE_API_KEY` missing from `.env` | Fill in `.env` |
| 403 on all endpoints | Application not subscribed to Légifrance API | PISTE portal → App → Edit → check "Légifrance v2.4.2" |
| `.env` not read | Windows saved file as `.env.txt` | Rename to `.env` (no extension) |

## 10. AN Roll-Call Vote Pipeline

A one-time build script downloads official Assemblée Nationale bulk data and produces a static `data/candidate_votes.json` lookup table — served as O(1) during web requests, with no repeated API calls at runtime.

### Data source

- **Scrutins ZIP:** `https://data.assemblee-nationale.fr/static/openData/repository/{leg}/loi/scrutins/Scrutins.json.zip`
  - Legislature 16 (2022–2024): 9.6 MB, 4,106 scrutins
  - Legislature 17 (2024–): 20.3 MB, 6,645 scrutins
- No authentication required (official open data)

### Running the pipeline

```bash
# Full build (writes data/candidate_votes.json)
python build_votes.py

# Discover scrutin IDs by keyword
python build_votes.py --discover "ukraine" --leg 16
python build_votes.py --discover "retraites" --leg 16

# Find a deputy's PA number by surname
python build_votes.py --find-pa "Rousseau" --leg 17

# Quick connectivity test
python build_votes.py --test
```
